Why Superscribe? Superscribe is not your average routing framework.

Hierarchical Routes

Superscribe routes are defined hierarchically for ease of maintenance and efficient matching

Re-usable Definitions

Each route segment definition is strongly typed... reduce code duplication by composing and re-using them however you like

Flexible Approach

Like to have your routes defined near your handlers? Not a problem. Like to define routes centrally all in one place? That's fine.

No Limits

Execute arbitrary code as part of your routing pipeline to match routes, make choices or return responses using our powerful DSL... the possibilities are endless!

Configure Web API


public static class WebApiConfig
{
public static void Register(HttpConfiguration config)
{
    config.Formatters.Remove(config.Formatters.XmlFormatter);

    SuperscribeConfig.RegisterModules(config);
}
}

Configure OWIN


public class Startup
{
public void Configuration(IAppBuilder app)
{
    var config = new SuperscribeOwinConfig();
    config.MediaTypeHandlers.Add(
        "text/html", 
        new MediaTypeHandler { Write = (res, o) => res.WriteAsync(o.ToString()) });

    app.UseSuperscribeModules(config);
}
}

Being Productive Superscribe makes routes easy to follow, understand and maintain

  • Native matchers Use built in functionality to define common routes
  • Hierarchical definitions Route definitions are stored and traversed as a tree or graph
  • Composing routes Create definitions from reusable, strongly typed nodes

Constant Matching


"api" / "values"

Regex Matching


"api" / ~"^[A-Za-z]+$"

Parameters


"api" / "values" / (ʃString)"Name"
"api" / "values" / (ʃBool)"Complete"

// Optional
"api" / "values" / -(ʃLong)"Id"

Controllers and Actions (Web API only)


// Regex direct controller and action name match
"api" / ʃ.Controller / ʃ.Action / -(ʃInt)"Id"

// Static match and controller name mapping
"api" / "blog" / "tags".Controller("BlogPostTags") / -(ʃInt)"Id"
"api" / "media" / "tags".Controller("MediaTags") / -(ʃInt)"Id"

// Override action selection when optional SubFields route is matched
"api" / "values".Controller("Values") / -"SubFields".Action("GetSubFields")
